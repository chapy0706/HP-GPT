# このリポジトリの憲法（短い）

このサイトは「index.html / style.css / script.js（＋JSONデータ）」で動く静的サイト。
AI（Claude Code / Codex）を使って編集するときは、速度よりも「現行挙動を壊さないこと」を優先する。

## 絶対に守ること

1. 既存の体験を破壊しない
   - イントロオーバーレイ → 「スキップ」 → TOP表示
   - 「はじまり」ボタン → チャットモーダルが開き、最初のステップが表示される
   - Feel（ロゴ）ページ → 100マスのグリッドが描画され、ロゴをクリックすると詳細が更新される

2. 変更は最小単位で
   - 目的に対して必要なファイルだけを編集する
   - まとめてリファクタしない（“ついで作業”は禁止）

3. 品質ゲートを通す
   - ローカルでもCIでも `make verify` を通す（E2Eが落ちたら直してから進む）
   - 証拠が必要なときは `make evidence`（`out/evidence/*.log`）

4. 秘密情報は触らない
   - `.env*` / 鍵 / トークン / 個人情報は「読まない・書かない・ログに出さない」

## 作業の型（SDD: 仕様駆動）

- 仕様の単一の真実（Single Source of Truth）は `docs/specs/`。
- 作業手順は `docs/runbook/`。
- 仕様にない変更をしたくなったら、まず spec に追記してから実装する。

## 入口コマンド

- `make install` 依存を入れる
- `make verify` 挙動破壊がないか検証
- `make evidence` 検証ログを保存
